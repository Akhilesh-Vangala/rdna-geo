# Checkpoints — How to Save and Restore Project State

This project uses **git** for checkpoints. Every checkpoint is a saved snapshot of all scripts, metadata, and documentation. You can go back to any checkpoint at any time without losing anything.

---

## How to Save a Checkpoint

Run this whenever you reach a milestone or before making changes:

```bash
cd /Users/akhileshvangala/Desktop/rna

git add -A
git commit -m "CHECKPOINT: describe what was done"
```

Examples:
```bash
git commit -m "CHECKPOINT: tools installed, metadata filled, pipeline ready"
git commit -m "CHECKPOINT: collaborator scripts added to variant_calling.sh"
git commit -m "CHECKPOINT: pipeline tested successfully on SRR6375927"
git commit -m "CHECKPOINT: all 54 samples processed"
```

---

## How to See All Checkpoints

```bash
git log --oneline
```

Example output:
```
a3f2c1d  CHECKPOINT: all 54 samples processed
b8e4f2a  CHECKPOINT: pipeline tested on SRR6375927
c1d9e3b  CHECKPOINT: collaborator scripts added
d4f1a2c  CHECKPOINT: tools installed, metadata filled, pipeline ready
e7b3c1a  CHECKPOINT: initial state — all setup docs and scripts
```

---

## How to Go Back to a Checkpoint

```bash
# See what changed since a checkpoint
git diff d4f1a2c

# Go back to a specific checkpoint (safe — creates new branch)
git checkout d4f1a2c

# Go back and make it the current state permanently
git reset --hard d4f1a2c
```

---

## What Gets Saved vs Not Saved

### Saved (versioned)
- All scripts (`.sh` files)
- All documentation (`.md` files)
- Metadata files (`sample_metadata.tsv`, `run_list.txt`)
- Configuration (`.gitignore`)

### NOT saved (too large / generated data)
- FASTQ files (raw sequencing data — downloaded fresh from NCBI)
- BAM files (generated by pipeline)
- VCF files (generated by pipeline)
- Reference genome (`.fa` files — get from Alka & Kinjal)
- Log files

Generated data is reproducible — just re-run the pipeline. Scripts and metadata are what matter.
